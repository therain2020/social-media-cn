# 内容模板库

## 小红书 · 笔记模板

### 模板 A：痛点解决型

适用场景：项目解决了一个明确的开发者痛点

```
📌 [标题：痛点 + 解决方案，≤20字]

[开头 Hook]
谁懂啊，[痛点描述，1 句白话] 😭

[背景]
[1-2 句解释为什么这是个问题，你的真实经历]

[解决方案]
✨ [核心功能 1] — [1 句说明]
✨ [核心功能 2] — [1 句说明]
✨ [核心功能 3] — [1 句说明]

[上手]
📍 第一步：[操作]
📍 第二步：[操作]
📍 第三步：[操作]
搞定 🎉

[使用感受]
[1-2 句真实体验，不是广告腔]

[CTA]
[项目名] 已经在 GitHub 开源了，链接放评论区～
有什么问题直接问我！

🏷️
#[标签1] #[标签2] #[标签3] #[标签4] #[标签5]
```

## 小黑盒 · 帖子模板

### 模板 D：技术分享型

```
标题：[开源/分享] [核心卖点一句话]

# 背景

[1 段，你为什么做这个项目 / 遇到了什么问题]

# 技术方案

[技术栈列表]
- 组网：[技术]
- 服务：[技术]
- 客户端：[技术]

[架构简述，1-2 段]

# 关键代码/配置

[代码块或配置片段，展示最关键的部分]

# 效果

[描述使用效果，有数据最好]

# 安装

```bash
[安装命令]
```

GitHub：[链接]

# 讨论

[1-2 个开放问题，邀请讨论]
```

### CTA 话术

| 场景 | 话术 |
|------|------|
| GitHub引流 | GitHub 链接放评论区了，觉得有用的话给个 Star ⭐ |
| 互动邀请 | 大家有类似的需求吗？评论区聊聊 |
| 求助扩散 | 觉得有用的话帮忙转发一下，让更多人看到 🙏 |
| 教程导流 | 完整教程在 GitHub README，每一步都写了 |

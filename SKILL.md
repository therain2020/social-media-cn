# 中文开发者社区推广 · Social Media CN

为开源项目/开发者工具在小红书和知识星球/小黑盒生成推广内容。

## 适用平台

| 平台 | 内容形式 | 目标受众 |
|------|---------|---------|
| **小红书** | 图文笔记 (标题 20 字 + 正文 + 话题标签) | 开发者、效率工具爱好者、技术学习人群 |
| **知识星球/小黑盒** | 社区帖子 (标题 + 正文 + 标签) | 硬核开发者、游戏/技术社区 |

## 工作流程

### Step 0：了解产品（必须）

在写任何推广内容之前，**必须**阅读项目代码和 README，提炼以下信息：

1. **一句话价值主张** — 这个项目解决了什么痛点？
2. **核心功能** — 3-5 个关键能力
3. **目标用户** — 谁会用它？为什么需要它？
4. **技术亮点** — 有什么独特的技术实现？
5. **使用门槛** — 几步能上手？

使用 `task` 工具派出子 Agent 阅读项目：

```
task(
  subagent_type="Explore",
  description="Analyze project for promotion content",
  prompt="Thoroughly explore [PROJECT_PATH] and extract: elevator pitch, target users, key features, tech highlights, setup difficulty. Report in Chinese for social media content creation."
)
```

### Step 1: 平台调研（推荐）

在写内容前，先在目标平台上搜索同类内容，了解什么风格受欢迎：

```
xhs search "[关键词]"      # 小红书搜索竞品内容
```

- 记录热门标题模式
- 观察高互动内容的正文结构
- 整理常用话题标签

### Step 2: 按模板生成内容 → `references/templates.md`

## 小红书模板

### 笔记结构

```
📌 标题 (≤20字，必须有吸引力，可用1个emoji)

正文：

[开头 Hook — 1-2 句，直击痛点或制造好奇]

[为什么重要 — 1 段，解释背景和问题]

[解决方案 — 2-3 段，核心卖点+使用体验，用emoji做视觉分隔]

[上手步骤 — 简洁的 numbered list]

[结尾 CTA — 邀请互动/关注/试用]

🏷️ 话题标签：
#tag1 #tag2 #tag3 ... (5-10 个)
```

### 小红书标题公式

| 类型 | 公式 | 示例 |
|------|------|------|
| 痛点型 | [痛点] + 解决方案 | 不想坐电脑前写代码？试试这个 |
| 结果型 | [数字] + [结果] | 5分钟把电脑变成随身开发环境 |
| 对比型 | 别再[旧做法]了，试试[新方案] | 别再买服务器了，手机就能远程开发 |
| 教程型 | [主题] 保姆级教程 | Claude Code 手机端使用保姆级教程 |
| 开源分享 | 开源一个[描述]的工具/项目 | 开源一个手机远程操控Claude Code的项目 |

### 小红书正文风格

- **调性**：个人化、有温度、真实体验感（不要官方腔、不要广告感）
- **长度**：200-800 字，短段落 (2-3 句一段)，善用空行
- **emoji 用法**：每段开头用 1 个 emoji 做视觉锚点，不要每句都用
- **展示感**：如果配图是终端截图，正文要解释图中在做什么
- **互动**：结尾留一个问题或邀请，引发评论

## 知识星球/小黑盒模板

### 帖子结构

```
标题：[类型标签] + 核心卖点

正文：

[背景 — 1 段，你是做什么的时候发现这个需求]

[技术方案 — 具体怎么实现的，用了什么技术栈]

[核心代码/架构 — 关键部分展示]

[效果对比 — 用前 vs 用后]

[开源地址 + 安装方式]

[讨论邀请 — 有什么改进建议？大家有类似需求吗？]

标签：#技术栈 #用途 #类型
```

### 小黑盒标题公式

| 类型 | 示例 |
|------|------|
| 技术分享 | [开源] 我把Claude Code装进了手机，通勤路上写代码 |
| 痛点解决 | 终于不用坐在电脑前等Claude跑完了 |
| 教程分享 | 手把手：Windows 一键配置SSH远程开发环境 |

### 小黑盒正文风格

- **调性**：技术硬核但不装逼、真诚分享、像老哥聊天
- **长度**：300-1000 字
- **必含**：技术栈列表、GitHub 链接、安装难度说明
- **加分**：踩坑记录、性能数据、对比表格

## 配图方案

> 配图用 `social-media-cn-image` 技能自动生成：HTML 可视化 + headless 浏览器截图 → PNG。详见该技能的 SKILL.md。

### 小红书配图（推荐 3-6 张，3:4 竖版 750×1000）

| 序号 | 内容 | 模板 |
|------|------|------|
| 1 | 封面图 | `cover-xhs`：大字标题 + accent 色条 + 纸色底 |
| 2 | 痛点描述 | `comparison`：用前 vs 用后双栏对比 |
| 3 | 核心功能截图 | `terminal`：深墨底 + accent 提示符 + 命令行 |
| 4 | 安装步骤 | `steps`：accent 编号 + 代码片段 |
| 5 | 效果展示 | `phone-text`：深色圆角矩形模拟手机界面 |
| 6 | CTA + GitHub 链接 | `cta-card`：accent 短线 + 排版引导 |

### 小黑盒配图（推荐 2-4 张）

| 序号 | 内容 | 模板 |
|------|------|------|
| 1 | 架构图 | `architecture`：CSS/SVG 系统架构流程图 |
| 2-3 | 关键代码/配置 | `code-block`：深色主题 + 行号 + 语法高亮 |
| 4 | 链接/二维码 | `cta-card`：GitHub 引导卡片 |

## 内容生成后的检查清单

生成每篇推广内容后，逐项检查：

- [ ] 标题是否 ≤20 字（小红书）/ 是否有吸引力（小黑盒）？
- [ ] 开头 3 秒内能抓住注意力吗？
- [ ] 正文有真实使用体验感吗？（不是产品说明书）
- [ ] 技术描述准确吗？（对照项目代码验证）
- [ ] 有 CTA 吗？（邀请互动/Star/试用）
- [ ] 话题标签选了 5-10 个热门+相关标签吗？
- [ ] 配图已用 social-media-cn-image 生成并截图了吗？
- [ ] 有 AI 味吗？如果有，跑一遍 humanizer-zh

## 使用 humanizer-zh 去 AI 味（强烈推荐）

生成中文内容后，必须用 `humanizer-zh` skill 检查并修复：

```
/humanizer-zh
```

重点检查：夸大象征意义、宣传性语言、以-ing结尾的浅层分析、模糊归因、破折号过度使用、三段式法则、AI词汇（赋能/打通/闭环/降本增效）、否定式排比、过多连接性短语。
